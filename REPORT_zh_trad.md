# TranscriptFlow 專案分析報告

## 概觀
**TranscriptFlow** 是一個彈性 AI 驅動的資料管線，用於將原始字幕（SRT）檔案轉換為可搜尋、已摘要且向量化的知識。它針對大規模的轉錄檔案（如 YouTube 頻道、播客、講座系列）提供生產就緒的工作流程，將非結構化文字轉變為檢索增強生成（RAG）資源。

## 與一般轉錄工具的獨特優勢
- **確定性的分塊邊界** – 避免 LLM 產生的時間戳幻覺。
- **原子化檢查點寫入** – 在崩潰或長時間執行後仍能安全恢復。
- **嵌入批次的斷路器保護** – 防止 API 成本失控。
- **稽核工具** – 檢測過時作業及跨檔案不一致性。
- **每個階段的可擴充併發控制**。

## 簡歷寫作建議（可直接引用的要點）
- 設計並實作了 **容錯 AI 資料管線**，處理超過 **10,000 小時** 的字幕資料並生成可搜尋的向量嵌入，將人工知識提取工作量降低 **90%**。
- 開發了 **Smart Merge 3.0 語意分塊演算法**，利用餘弦相似度與百分位閾值，比固定長度視窗提升約 **15%** 的分塊相關性分數。
- 建構了 **看門狗編排系統**，實現自動階段推進與逾時作業恢復，達成 **24/7** 不間斷批次處理。
- 實作了 **統一的 LLM 客戶端**，支援多家供應商並具指數退避重試與模型回退策略。
- 引入 **細粒度診斷與模型效能指標**，協助基於資料進行模型選型與成本最佳化。
- 提供 **完整的測試套件**，涵蓋分塊品質、摘要忠實度與 SRT 完整性，綜合通過率超過 **95%**。
